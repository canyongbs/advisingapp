findUserSubscription

Description

Get a subscription by its primary key.

Response

Returns a UserSubscription

Arguments
Name Description
id - ID! The primary key of the subscription.

Example

Query
query FindUserSubscription($id: ID!) {
  findUserSubscription(id: $id) {
    id
    user {
      id
      email
    }
    subscribable {
      ... on Prospect {
        ...ProspectFragment
      }
      ... on Student {
        ...StudentFragment
      }
    }
  }
}
Variables
{"id": "4"}
Response
{
  "data": {
    "findUserSubscription": {
      "id": 4,
      "user": User,
      "subscribable": Prospect
    }
  }
}

Queries

userSubscriptions

Description

Get all subscriptions.

Response

Returns a UserSubscriptionPaginator!

Arguments
Name Description
first - Int! Limits number of fetched items. Maximum allowed value: 100. Default = 25
page - Int The offset from which items are returned.

Example

Query
query UserSubscriptions(
  $first: Int!,
  $page: Int
) {
  userSubscriptions(
    first: $first,
    page: $page
  ) {
    paginatorInfo {
      count
      currentPage
      firstItem
      hasMorePages
      lastItem
      lastPage
      perPage
      total
    }
    data {
      id
      user {
        ...UserFragment
      }
      subscribable {
        ... on Prospect {
          ...ProspectFragment
        }
        ... on Student {
          ...StudentFragment
        }
      }
    }
  }
}
Variables
{"first": 25, "page": 987}
Response
{
  "data": {
    "userSubscriptions": {
      "paginatorInfo": PaginatorInfo,
      "data": [UserSubscription]
    }
  }
}
Queries

users

Description

List multiple users.

Response

Returns a UserPaginator!

Arguments
Name Description
name - String Filters by name. Accepts SQL LIKE wildcards % and _.
first - Int! Limits number of fetched items. Maximum allowed value: 100. Default = 10
page - Int The offset from which items are returned.

Example

Query
query Users(
  $name: String,
  $first: Int!,
  $page: Int
) {
  users(
    name: $name,
    first: $first,
    page: $page
  ) {
    paginatorInfo {
      count
      currentPage
      firstItem
      hasMorePages
      lastItem
      lastPage
      perPage
      total
    }
    data {
      id
      email
    }
  }
}
Variables
{"name": "xyz789", "first": 10, "page": 987}
Response
{
  "data": {
    "users": {
      "paginatorInfo": PaginatorInfo,
      "data": [User]
    }
  }
}

Types

ID

Description

The ID scalar type represents a unique identifier, often used to refetch an object or as key for a cache. The ID type appears in a JSON response as a String; however, it is not intended to be human-readable. When expected as an input type, any string (such as "4") or integer (such as 4) input value will be accepted as an ID.

Example
4
Types

Int

Description

The Int scalar type represents non-fractional signed whole numeric values. Int can represent values between -(2^31) and 2^31 - 1.

Example
987
Types

PaginatorInfo

Description

Information about pagination using a fully featured paginator.

Fields
Field Name Description
count - Int! Number of items in the current page.
currentPage - Int! Index of the current page.
firstItem - Int Index of the first item in the current page.
hasMorePages - Boolean! Are there more pages after this one?
lastItem - Int Index of the last item in the current page.
lastPage - Int! Index of the last available page.
perPage - Int! Number of items per page.
total - Int! Number of total available items.
Example
{
  "count": 987,
  "currentPage": 987,
  "firstItem": 987,
  "hasMorePages": false,
  "lastItem": 987,
  "lastPage": 987,

  "total": 987
}
